Sara2lvex



Sara2lvex is a new program now available for beta testing. Sara2lvex reads VLBA log files in the format produced by the NRAO program sara and writes Mark-4-compatible lvex files. Sara2lvex needs to read also the ovex file for the same experiment.

Usage:

sara2lvex sarafilename ovexfilename > outfilename

The quality of the lvex output is dependent on the quality of both the input files: You need to have a good ovex file before running sara2lvex.



Sara is available on various NRAO machines such as aspen.aoc.nrao.edu. On such machines, sara is available in /opt/local/vlbsoft/bin/sara with source code in /home/magnolia2/vlbsoft/obsprgms/solaris/sara.c (etc.) and documentation in /home/magnolia2/vlbsoft/doc/sara.txt. Sara reads VLBA binary logs and writes an ASCII equivalent, which, depending on the commands to sara, can contain all the information in the binary logs. Sara2lvex works well with the sara defaults, or, to save a little disk space, ask for typecodes C1, COM, FLG, and TAP only.



Sara2lvex has some known problems especially with respect to the notorious broken scans: VLBA stations usually ignore the pass numbers and tape directions in the schedule and put scans on the tape wherever they fit. This results in some scans being broken into two pieces when the tape drive hits either of the low-tape sensors. Sara2lvex makes such broken scans into two partial scans (with the same scan name) and adds a comment "Continuation of previous scan" to the second part. Search for these comments with an editor. Deleting or commenting out the less-desirable of the partial scans is probably the best that can be done with the Mark-4 correlator: Genaroot will use the last of the two if it is not deleted or commented.



Sara2lvex is based on a Perl script, vlvex.pl, written by Dave Graham of MPIfR, Bonn, Germany and on a C program, vlogx, written by Nancy Vandenburg of NVI, Inc. and NASA GSFC. Vlogx is in ftp://gemini.gsfc.nasa.gov/pub/sked/vlogx/source/... with documentation in .../pub/sked/vlogx/vlogx.txt.
