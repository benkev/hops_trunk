Mark 4 Correlator Error Explanations

0: Error number 0 is not defined. You are on your own for this one.

1: The building is on fire. Please shut down the correlator in an orderly fashion before saving yourself.

1002: Opera was unable to open and read the named task-stream (.tsf) file. Did you spell it correctly? Is it really in that directory?

1004: Opera was unable to open and write to the named task-stream (.tsf) file. Did you spell it correctly? Is it really in that directory? Do you have write permission?

2000: The correlator control manager, corrman, has rejected a configuration request. The possible errors are encoded within the return_code; see the Cu_Resource_Error structure in message_structs.h for details.

2001: The station unit manager, suman, was unable to be configure the indicated station unit. The return_code denotes the cause.

3000: Suman was unable to initialize the indicated station unit. If re-initialization doesn't correct the situation, then the SU may need to be re-booted.

6000: Sutrx has encountered a non-recoverable socket error for the indicated SU. Quite likely the cause is flaky SU code that has decided to go on holidays. If this station unit is needed, it may be necessary to exit the correlator software, reboot the SU, and start the software up again.
